# Quickstart: Validating the Flora Pastel Marketing Website

How to prove this feature works end-to-end once implemented. This is a validation guide, not an implementation guide — see `tasks.md` (generated by `/speckit-tasks`) for build steps.

## Prerequisites

- Repository dependencies already installed (spec 002 set up `package.json`).
- Business Profile content present in `src/content/business.ts`, reconciled against `cliente/datos.md`.
- At least one Custom Cake example present under `src/content/custom-cakes/` per `contracts/custom-cake-content-schema.md`.

## Build and preview

```bash
npm run build
npm run preview
```

Expected: build completes with no Content Collection schema errors, and `dist/sitemap-index.xml` and `dist/robots.txt` exist (FR-019, FR-020).

## Scenario walkthroughs (map to spec User Stories)

### US1 — Discover the brand and order a signature cake via WhatsApp

1. Open `/`. Confirm a hero section communicates the brand and a prominent WhatsApp CTA is visible without scrolling on mobile (FR-004).
2. Confirm the page links to `/catalogo` (Signature Cakes, superseded per `research.md`) and to `/pasteles-personalizados` (Custom Cakes) — FR-005.
3. Follow the `/catalogo` link and confirm the site-wide nav/footer now appears there too (retrofitted `BaseLayout`).
4. Tap the WhatsApp CTA and confirm it opens a pre-filled conversation (FR-011, FR-012).

**Automated coverage**: `tests/e2e/home.spec.ts`, `tests/e2e/site-chrome.spec.ts`.

### US2 — Request a custom cake for a celebration or corporate event

1. Navigate to `/pasteles-personalizados`.
2. Confirm a gallery of past custom work renders, each item with a photo, title/description, and occasion/style tag (FR-008).
3. Use an item's WhatsApp CTA and confirm the pre-filled message identifies the request context (FR-009, FR-012).
4. Confirm there is no structured multi-field intake form — only the WhatsApp CTA (FR-027).

**Automated coverage**: `tests/e2e/custom-cakes.spec.ts`.

### US3 — Find business information and location before ordering or visiting

1. Navigate to `/contacto`.
2. Confirm the address, phone number, and hours of operation are visible (FR-013).
3. Confirm the embedded map renders, and that the text address + "Cómo llegar" directions link are visible independent of the map's load state (FR-014, Edge Cases).
4. Confirm social media links are present, both here and in the site-wide footer (FR-015).

**Automated coverage**: `tests/e2e/contact.spec.ts`.

### Site-wide chrome and 404

1. Visit `/`, `/catalogo`, `/pasteles-personalizados`, `/contacto` and confirm the same nav (with links to all four) and footer render on every one (FR-001, FR-002, FR-003).
2. Visit a nonexistent path (e.g., `/no-existe`) and confirm the custom 404 page renders with a link back to Home and a WhatsApp/contact path (FR-021).

**Automated coverage**: `tests/e2e/site-chrome.spec.ts`, `tests/e2e/not-found.spec.ts`.

## Resilience checks

- **No-JS check**: Re-run the US1–US3 walkthroughs with JavaScript disabled. Every piece of business-critical content (branding, photography/descriptions, address, phone, hours, social links, and all WhatsApp CTAs) must remain visible and usable (FR-024).
- **Accessibility check**: Run `@axe-core/playwright` against `/`, `/pasteles-personalizados`, and `/contacto`; zero critical/serious violations (constitution WCAG AA target), matching the zero-violation result already achieved for `/catalogo`.
- **Performance check**: Run Lighthouse (mobile + desktop) against all four pages; Performance/Accessibility/Best Practices/SEO each ≥ 90 (SC-002, SC-003).
- **SEO check**: Confirm each page has a unique `<title>`/description, valid Open Graph tags (spot-check with a link-preview tool), and a `LocalBusiness` JSON-LD block that validates against schema.org (FR-016, FR-017, FR-018). Confirm `sitemap-index.xml` lists all four pages and `robots.txt` references it.

## Done when

- All `tests/e2e/{home,custom-cakes,contact,site-chrome,not-found}.spec.ts` suites pass, including their no-JS variants.
- Lighthouse and axe checks meet the thresholds above on every page, including `/catalogo` after the nav/footer retrofit.
- The Business Profile in `src/content/business.ts` is reconciled line-by-line against `cliente/datos.md`.
